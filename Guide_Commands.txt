🐍 Python 批量重命名项目：环境配置与打包全攻略
第一阶段：初始化虚拟环境 (仅在新项目开始时执行一次)
如果你删除了旧环境想重新开始，或者在新的电脑上同步了代码：

PowerShell
# 1. 确保你在项目根目录 (C:\Users\Ritchie\Documents\ReName-File-Name)
# 2. 创建名为 .venv 的虚拟环境文件夹
python -m venv .venv
第二阶段：激活环境与安装工具 (每次关闭并重启 IDE 后执行)
在编写代码或打包前，必须先“进入”这个小房间。

PowerShell
# 1. 允许当前窗口运行脚本（如果报错“禁止执行脚本”才需要跑这行）
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process

# 2. 激活 .venv 环境
.\.venv\Scripts\Activate.ps1
# 成功标志：命令行开头会出现 (.venv) 字样
退出虚拟环境：deactivate

# 3. 在这个独立环境下安装打包工具
pip install pyinstaller
第三阶段：运行与调试 (在 Antigravity 中反复测试)
在打包成 EXE 之前，直接运行脚本确认逻辑正确。

PowerShell
# 运行脚本
python smart_rename.py
测试建议：

正则表达式：.*_([a-z]{3})_(\d+)_(\d{4})\.pdf

目标格式：房贷-yyyy-mm-dd.pdf 或 房贷-yyyy-mmm-dd.pdf

第四阶段：生成自执行文件 (.exe)
当你确认程序在 Antigravity 里运行完美后，生成免安装版。

PowerShell
# 生成单个独立的可执行文件
# --onefile: 全部打包成一个文件
# --console: 运行时显示命令行窗口（用于输入路径和正则）
pyinstaller --onefile --console smart_rename.py
第五阶段：产出物说明
EXE 位置：打包完成后，去 dist/ 文件夹里找 smart_rename.exe。

可移动性：你可以把这个 .exe 拷贝到任何没有安装 Python 的 Windows 电脑上直接运行。

清理：打包成功后，build/ 文件夹和 .spec 文件可以随时删除。

💡 常见问题排查
找不到 pyinstaller 命令：请检查命令行开头是否有 (.venv)，如果没有，请重新执行“激活环境”的步骤。

文件名含有斜杠：程序已内置保护，会自动把 mm/dd/yyyy 中的 / 替换为 -。

关于正则表达式：
在正则表达式中，点号 . 是最强大但也最需要小心使用的元字符（Metacharacter）。
在你的文件名处理场景中，它主要扮演“全能替身”和“贪婪填充”的角色。
1. 点号 . 的核心含义在正则语法中，. 代表 “匹配除换行符以外的任何单个字符”。它能匹配字母、数字、空格、
下划线、连字符、括号，甚至是特殊符号（如 @, #, $）。请注意：是单个字符！
(.*_)写法：配合 *（代表匹配 0 次或多次）；
2. 在你的文件名规则中，. 的两种用法在你刚才那个复杂的文件名 line_of_crediet_-_..._AUg_31_2024 - 
Copy (3).txt 中，我们用了两次点号，它们的作用各不相同：
A. 作为“前置过滤器” (.*_)写法：.*_作用：这里的 . 配合 *（代表匹配 0 次或多次），意味着**“吃掉下划线之
前的所有乱七八糟的内容”**，直接精准定位到后面我们关心的月份
数据。
B. 作为“后缀吸收器” (.*\.)写法：.*\.作用：匹配年份之后到扩展名点号之前的所有内容。
解决的问题：专门用来对付文件名末尾那些烦人的  - Copy,  (1), _v2 等多余后缀。
它把这些内容匹配到，但不提取，从而让我们跳过它们直接拿到扩展名。
3. 特别注意：点号 . 与 转义点 \. 的区别这是很多开发者最容易混淆的地方，
在你的代码里这两者都出现了：符号名称作用例子.元字符 (Dot)匹配任意一个字符a.c 
\.转义点 匹配实际意义的'.'
